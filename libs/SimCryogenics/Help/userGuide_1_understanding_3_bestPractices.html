1.3. Best practices 

a. Parameters

This rules works for all Simulink or Simscape schema. You shouldn't enter the value of a parameter or an input directly in a block.
You should create a .m file that contains all the parameters and inputs of your schema, and implement the name of this parameters/inputs in the schema.

b. Solver

The recommended solver for Simscape is currently ode15s. You should use this one, or the one that Simscape recommended to you.

c. Fluid properties

When you want to change the parameter of fluid properties, you shouldn't set a very large range of value for the pressure or temperature.
If the minimum and maximum value are too spread, the precision of the result should decline.
So be sure that your range is not to large for the applications you want to test.

Another thing about this block is the fact that each of your model should implement it.
You can't run a model without this block. You just have to connect it anywhere in the schema.
